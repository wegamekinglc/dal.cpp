# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build Commands

```bash
# Full build (includes code generation, XAD, and tests)
./build_linux.sh

# Manual build
mkdir build && cd build
cmake --preset=Release-linux .. && make -j32 && make install

# Debug build
cmake --preset=Debug-linux .. && make -j32 && make install
```

CMake installs to the repo root (`CMAKE_INSTALL_PREFIX=${sourceDir}`), placing binaries in `bin/` and headers in `include/`.

## Running Tests

```bash
# Run all tests
bin/test_suite

# Run a single test suite
bin/test_suite --gtest_filter=<SuiteName>.*

# Run a single test
bin/test_suite --gtest_filter=<SuiteName>.<TestName>
```

## Code Style

Formatting is enforced via `.clang-format` (LLVM-based):
- 4-space indentation, 150 column limit
- `PointerBindsToType: true` (use `T*` not `T *`)
- Braces attach style (`BreakBeforeBraces: Attach`)

## Architecture

This is a C++17 quantitative finance library with AAD (Automatic Adjoint Differentiation) support via the XAD framework.

**Core library (`dal/`)** — built into `libdal_library`:
- `math/` — numerical algorithms: interpolation, optimization, PDE solvers, random number generation, matrix ops, root finding
- `script/` — expression scripting engine using visitor pattern (parser → AST nodes → simulation/evaluation)
- `model/` — financial models
- `curve/` — yield/discount curve handling
- `indice/` — reference rate index management
- `risk/` — risk calculations
- `concurrency/` — thread pool and concurrent queue
- `storage/` — data persistence (files matching `_repository.*` are excluded from the build)
- `auto/` — auto-generated code (excluded from build, generated by Machinist)

**Public API (`public/`)** — wraps the core library for external consumption. Python bindings and Excel integration are planned but not yet implemented.

**Code generation** — `config/dal.ifc` is processed by the Machinist tool to generate files in `dal/auto/` and `public/auto/`. Run `build_linux.sh` to regenerate.

**External dependencies** (`external/`):
- `xad/` — XAD AAD framework for automatic differentiation
- `googletest-1.17.0/` — test framework
- `rapidjson/` — JSON parsing
- `machinist/` — code generation tool

**Tests (`tests/`)** — one subdirectory per module, all compiled into a single `test_suite` binary using Google Test.

**Examples (`examples/`)** — standalone programs demonstrating AAD, Monte Carlo, finite difference, and scripting usage.
